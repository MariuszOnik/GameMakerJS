🔍 Inspektor (Panel dolny – Scena)

Po kliknięciu obiektu w inspektorze możesz edytować jego właściwości.

NazwaIdentyfikator obiektu – używany w węzłach logiki jako "Sprite ID".
X / YPozycja obiektu na scenie (środek obiektu).
W / HRozmiar w pikselach (szerokość / wysokość).
ObrazPrzypisuje teksturę ze zbioru Zasobów do tego sprite'a.
⧉ DuplikujKopiuje obiekt z przesunięciem +24px.
🗑 UsuńUsuwa obiekt ze sceny (lub klawisz Delete / Backspace).

Siatka / Snap (gdy nic nie zaznaczone) – włącz snapowanie do siatki i ustaw X/Y kroku.

💡 Kliknij „▲ Panel" żeby zwinąć / rozwinąć inspektor i zyskać więcej miejsca na scenę.

⚡ Fizyka (sekcja w inspektorze)

Fizyka działa tylko w trybie Gra (▶). W edytorze obiekty są statyczne.

Fizyka ✓Włącza silnik Arcade Physics na tym obiekcie.
Statyczny ✓Obiekt nie porusza się – idealny na platformy i podłogę.
Odbicie0 = brak odbicia, 1 = pełne odbicie (piłka). Ułamki pomiędzy.
Grawitacja ✓Czy grawitacja (y = 500 px/s²) ciągnie obiekt w dół.
Granice ✓Obiekt nie wychodzi poza krawędzie ekranu.
Kamera śledzi ✓Kamera podąża za tym obiektem podczas gry.

💡 Collider (prostokąt kolizji) ma rozmiar W × H obiektu. Kolizje między obiektami fizycznymi są automatyczne – nie trzeba nic łączyć węzłami.

🖼 Zasoby (przycisk 🖼 w nagłówku)

Przechowuje obrazy PNG/JPG/GIF dostępne dla wszystkich projektów.

Wgraj obrazWczytuje plik z dysku. Obraz trafia do pamięci przeglądarki.
➕ Do scenyTworzy sprite i od razu przypisuje ten obraz.
✕ na miniaturzeUsuwa zasób (nie usuwa obiektów w scenie).

💡 W inspektorze kliknij „Obraz → Brak – wybierz…" żeby otworzyć Zasoby w trybie pickera i przypisać obraz do wybranego sprite'a.

⬡ Zakładka Logika – Węzły

Graf węzłów definiuje zachowanie gry. Węzły łączą się portami – białe = exec (przepływ), żółte = liczba, fioletowe = tekst.

＋ WęzełOtwiera menu z listą węzłów. Wpisz nazwę żeby filtrować. Enter dodaje pierwszy wynik.
PołączeniePrzeciągnij z portu wyjściowego na port wejściowy tego samego koloru.
Usuń połączenieKliknij na linię połączenia – zmieni kolor na czerwony, kliknij ponownie żeby usunąć.
✕ na węźleUsuwa węzeł wraz z jego połączeniami.
Pan / ZoomPrzeciągnij tło żeby przesunąć widok, scroll żeby zbliżyć/oddalić.

Kategorie węzłów:

🟢 ZdarzeniaNa Start – uruchamia się raz; Na Update – każda klatka; Na Dotyk/Klik – dotknięcie ekranu (daje X, Y).
⚡ AkcjeRusz Sprite (delta dX/dY), Ustaw pozycję (bezpośrednie X/Y), Ustaw Prędkość (fizyka VX/VY), Skocz, Pokaż/Ukryj (pokaż/ukryj/przełącz), Wyświetl Tekst, Ustaw Zmienną, Warunek, Czekaj, Log.
🔢 WartościLiczba, Tekst, Matematyka (+−×÷%), Losowa liczba, Pobierz Zmienną, Pobierz właściwość (x/y/vx/vy/width/height obiektu).

💡 "Sprite ID" w węzłach = pole Nazwa obiektu z inspektora sceny (np. Sprite1, Gracz).

</> Własne węzły – krok po kroku

Kliknij przycisk </> w toolbarze Logiki. W edytorze kodu wpisz definicję węzła i kliknij ⚡ Zarejestruj węzeł. Węzeł pojawi się w menu ＋ Węzeł i jest gotowy do użycia w grafie.

Krok 1 – tożsamość węzła

registerNode({
  type: 'dodaj-punkty',   // unikalny klucz – nie używaj spacji ani polskich liter
  label: 'Dodaj punkty',  // nazwa wyświetlana w menu
  icon: '⭐',             // emoji widoczne na węźle
  category: 'action',     // 'action' (robi coś) | 'value' (zwraca wartość) | 'event'

Krok 2 – właściwości (props) – pola widoczne bezpośrednio na węźle.

  props: {
    // Zwykłe pole tekstowe:
    name: { label: 'Zmienna', defaultValue: 'punkty' },

    // Lista rozwijana:
    mode: { label: 'Tryb', defaultValue: 'dodaj', options: ['dodaj', 'odejmij', 'ustaw'] },

    // Liczba:
    amount: { label: 'Ilość', defaultValue: 10 },

    // Lista obiektów sceny (zamiast wpisywania nazwy ręcznie):
    target: { label: 'Sprite ID', defaultValue: 'Sprite1', inputType: 'sprite-select' },
  },

Krok 3 – porty wejściowe (opcjonalne) – pozwalają podłączyć węzły Liczba / Pobierz Zmienną itp.

  inputs: [
    // type: 'number' lub 'string'
    { id: 'amount', label: 'Ilość', type: 'number' },
  ],
  // Uwaga: port 'exec' jest dodawany automatycznie – nie wpisuj go tutaj.

Krok 4 – funkcja run(inputs) – zwraca fragment kodu JS który zostanie wstrzyknięty do gry.

  run(inputs) {
    // inputs.X  →  gotowe wyrażenie JS dla danego portu lub prop
    //
    // Jeśli prop 'target' = 'Gracz'  →  inputs.target  =  "Gracz"
    // Jeśli port 'amount' podłączony do węzła Liczba(5)  →  inputs.amount  =  5
    // Jeśli port 'amount' podłączony do węzła Zmienna  →  inputs.amount  =  (this.variables.get("x") ?? 0)
    //
    // Wewnątrz szablonu (backtick) wstawiaj inputs.X przez ${inputs.X}

    return `
      // 'this' = PlayScene Phaser – masz dostęp do:
      // this.sprites      – Map
      // this.variables    – Map
      // this.time         – timery: this.time.delayedCall(500, () => { ... })
      // this.cameras.main – kamera: zoom, follow, shake, fade
      // this.physics      – silnik fizyki Arcade

      const _punkty = (this.variables.get(${inputs.name}) ?? 0);
      this.variables.set(${inputs.name}, _punkty + ${inputs.amount});
    `
  }
})

Pełny przykład – węzeł "Dodaj punkty":

registerNode({
  type: 'dodaj-punkty',
  label: 'Dodaj punkty',
  icon: '⭐',
  category: 'action',
  inputs: [
    { id: 'amount', label: 'Ilość', type: 'number' }
  ],
  props: {
    name:   { label: 'Zmienna',  defaultValue: 'punkty' },
    hudId:  { label: 'HUD ID',   defaultValue: 'Tekst1', inputType: 'sprite-select' },
  },
  run(inputs) {
    return `
      const _v = (this.variables.get(${inputs.name}) ?? 0) + ${inputs.amount};
      this.variables.set(${inputs.name}, _v);
      const _hud = this.sprites.get(${inputs.hudId});
      if (_hud && _hud.setText) _hud.setText('Punkty: ' + _v);
    `
  }
})

💡 Kliknij przycisk { } w toolbarze Logiki → podgląd wygenerowanego JS z całego grafu. Przydatne gdy coś nie działa – zobaczysz dokładnie co zostało skompilowane.

💡 Węzły są zapisane w przeglądarce (localStorage). Kliknij ✏ na liście by edytować, 🗑 by usunąć (usuwa też wszystkie instancje z grafu).
